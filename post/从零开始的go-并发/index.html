从零开始的Go(?)-并发

Mar 05, 2023
阅读时长: 2 分钟

概念

对于一些关于进程、线程、协程以及并发与并行的概念，在这里就不具体提了(其实我也不太搞得清楚)。

可以参考“Go入门指南"的第十四章

Goroutine

goroutine是由Go运行管理的轻量级线程(协程)，Go可以调用多个goroutine来充分调用系统资源，来实现高速的运算。

认识Goroutine

通过一个简单的例子来认识如何使用goroutine。

package main

import (
	"fmt"
)

func hello(){
    fmt.Println("Hello")
}

func main() {
    go hello()	//使用goroutine来调用hello函数
    fmt.Println("1")
}

一个最简单的goroutine就实现了，但是当你运行这段代码时会发现问题，输出的结果为1，并没有输出go hello()语句。

这是因为与其他函数不同，go在goroutine执行结束前会直接跳到下一行运行，可能导致主函数(主函数也可以理解为主goroutine，而其他goroutine为子goroutine)已经结束，程序已经终止了，自然不会返回goroutine的语句。

通过time.Sleep()函数可以简单的解决(但并不优雅)

package main

import (
	"fmt"
    "time"
)

func hello(){
    fmt.Println("Hello")
}

func main() {
    go hello()	//使用goroutine来调用hello函数
    fmt.Println("1")
    time.Sleep(1 * time.Second) //使程序在结束之前先睡眠1s
}

通过加上time.Sleep(1 * time.Second)语句会让程序在结束之前先行等待，从而让子goroutine返回。

WaitGroup

虽然time.Sleep()可以解决上面的问题，但你可能意识到了：我怎么知道goroutine在多久后结束？而通过sync包中的sync.WaitGroup便可以实现goroutine的同步。

package main

import (
	"fmt"
	"sync"
)

var wg sync.WaitGroup //WaitGroup就相当于一个计数器，默认值为0

func hello(i int) {
	defer wg.Done() // 在每个goroutine执行完成后使计数器-1
	fmt.Printf("Hello %d\n", i)
}

func main() {
	for i := 0; i < 10; i++ {	
		wg.Add(1) //使计数器+1
		go hello(i)
	}
    wg.Wait() //使程序阻塞，直到计数器为0(也就是所有子goroutine完成)
}

信道(通道) Channel

Go提倡通过通信来实现共享内存，而不是通过共享内存来实现通信

而根据这一原则，我们在使用goroutine时尽量不要使用全局变量，会产生大量问题，而为了在不同的goroutine之间实现共享内存，通过信道便可以实现通信。

创建信道

1
2

var 信道实例 chan 信道传输的数据类型
var chan1 chan int

通过var 语句可以声明一个信道，但是其初始值为nil,需要用make函数将其初始化

1

chan1 = make(chan int)

当然，也可以将声明与初始化合并这样写：

1

chan1 := make(chan int)

数据传输

对于信道我们可以传入数据，也可以读取数据。

1
2

chan1 <- 100 // 向chan1信道中传入100
a := <- chan1 //从chan1信道中读取数据，得到100

关闭信道

通过close()函数可以将信道关闭。在关闭之后，依然可以从信道中读取数据，但是接受到的值永远会是0。

1
2

chan1 := make(chan int)
close(chan1)

对于信道是否关闭，go提供了多返回值来判断：

1
2
3

chan1 := make(chan int)
close(chan1)
a,ok := <-chal	//ok的值为false,表示信道已关闭，若为true则为开

缓冲

在创建信道时，我们还可以设定信道的容量

1
2

chan1 := make(chan int) //不填信道的容量默认为0
chan2 := make(chan int,10) //设定信道的容量为10

通过cap()函数可以得出信道的容量，而len()函数返回信道中当前的数据个数。

我们将容量为0的信道称为无缓冲信道，而将容量大于0的信道称为有缓冲信道。

无缓冲信道

对于无缓冲信道，发送数据与接受数据是同时进行的。

package main

import "fmt"

func sum(s []int, c chan int) {
	sum := 0
	for _, v := range s {
		sum += v
	}
	c <- sum // 将和送入 c
}

func main() {
	s := []int{7, 2, 8, -9, 4, 0}
	c := make(chan int)
	go sum(s[:len(s)/2], c)	// -5
	go sum(s[len(s)/2:], c)	// 17
	x, y := <-c, <-c // 从 c 中接收，接收的顺序与传入的顺序相同,传入的协程同步进行，并不知道哪个先完成，比较随机
	fmt.Println(x, y) 
}

package main

import "fmt"

func sum(s []int, c chan int) {
	sum := 0
	for _, v := range s {
		sum += v
	}
	c <- sum // 将和送入 c
}

func main() {
	s := []int{7, 2, 8, -9, 4, 0}

	c := make(chan int)
	go sum(s[:len(s)/2], c)	// -5
	go sum(s[len(s)/2:], c)	// 17
	x, y := <-c, <-c // 从 c 中接收，接收的顺序与传入的顺序相同

	fmt.Println(x, y, x+y) // -5 17 12
}
